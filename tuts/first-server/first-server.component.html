Install the plugin

So, first, when you already have some servers configured in the config.yml of your BungeeCord server, they will be existing, even when you have the ServerManager plugin installed.

But, I recommend that you setup at least one lobby through the ServerManager plugin, so you can make sure the forceToLobby setting works and you can use the load balancing feature of the plugin.

You can also remove all servers from the config.yml of your BungeeCord server. The servers you add through ServerManager will appear as the normal servers.

The easiest way to install you first server is if you already have a server installed on your BungeeCord server, join the network and execute the following command:

/sm create [servername] [ip-address] [port] [access level] [lobby true / false]
Explanation:

[servername]: The name your new server's going to have.
[ip-address]: The IP-Address of your new server.
[port]: The port of your new server.
[access level]: You can choose an access level for your server, if you choose all, every player is able to join.
[lobby]: You can set this value to true or false, it sets, whether this server is going to be a lobby or a non-lobby server.

By default, the maximum amount of players, that can join a server is at 20 players.
You can change that with that command:

/sm setmaxplayer [servername] [new amount of players]
Explanation:

[servername]: The name of the server you want to edit.
[amount of players]: The name of the server you want to edit.

Hints:

First: You have to have at least one lobby registered via the ServerManager plugin and this lobby. If there is only one lobby, it should have the access level all.

Second: All players that don't have a lobby to join on are going to be kicked. This includes new players that don't have the access to join a lobby.

Third: If you want to use the WebEditing feature of the plugin, read that first: Beam me there
